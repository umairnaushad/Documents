Azure AD is not a cloud version of Windows Server Active Directory
 A tenant represents the organization and the default directory assigned to it.
 Subscriptions in Azure are both a billing entity and a security boundary
 Each subscription also has a single account owner
 subscription billing can be transfered to another account 
 One user can be in multiple Azure AD
 Azure AD region cannot be changed later
 Azure AD Connect is a separate service that allows you to synchronize a traditional Active Directory with your Azure AD instance
  Azure AD administrator can create security group
  Dynamic user assignment in a group require Azure AD Premium P1 license
 Custom role creation requires Azure AD Premium P1 or P2 and cannot be done in the free tier.
 Azure AD Connect replaces older versions of identity integration tools, such as DirSync and Azure AD Sync

 Azure AD Roles: (Global Administrator, User Administrator, Billiing Administrator)
 RBAC Roles: (Owner, Contributor, Reader, User Access Administrator)
Azure RBAC roles apply to Azure resources, and Azure AD roles apply to Azure AD resources (particularly users, groups, and domains)
By default, a Global Administrator doesn't have access to Azure resources.
